# Telemt Manager

Менеджер для установки, обновления и удаления [Telemt](https://github.com/telemt/telemt) — Telegram-прокси — через systemd на Debian-based дистрибутивах Linux (Ubuntu, Mint, Kali и др.).

## Установка

```bash
bash <(wget -qO - https://raw.githubusercontent.com/DenisShahbazyan/Telemt-Manager/master/telemt-manager.sh)
```

## Меню

```
Главное меню
├── 1) Установить / Переустановить
│   ├── 1) Автоматическая установка
│   ├── 2) Ручная установка
│   └── Enter) Назад
├── 2) Обновить
├── 3) Удалить
├── 4) Пользователи
│   ├── 1) Добавить пользователя
│   ├── 2) Редактировать пользователя
│   ├── 3) Удалить пользователя
│   └── Enter) Назад
├── 5) Редактировать конфиг
└── Enter) Выход
```

## Режимы установки

### Автоматический

Установка с настройками по умолчанию:

| Параметр | Значение |
|---|---|
| Порт | `443` |
| TLS домен | `github.com` |
| Имя пользователя | `user1` |
| Секрет | генерируется автоматически (`openssl rand -hex 16`) |

### Ручной

Установка с выбором параметров вручную:

- **Порт** — проверяется, что указанный порт свободен
- **TLS домен** — произвольный домен для маскировки трафика
- **Имя пользователя** — произвольное имя
- **Секрет** — автогенерация или ввод вручную (ровно 32 hex-символа)

> Если на сервере уже есть конфиг (`/etc/telemt/telemt.toml`), оба режима предложат использовать его повторно.
